# 🍷 Wine Casino

Онлайн-приложение для игры в «Винное казино». Организатор настраивает вина с правильными ответами, игроки делают ставки по категориям со своих телефонов.

## Файлы

- `index.html` — приложение (на GitHub Pages)
- `favicon.svg` — иконка
- `setup.sql` — SQL для Supabase (один раз)

## Деплой

1. Загрузи `index.html` и `favicon.svg` в репозиторий
2. Settings → Pages → Source: main → Save
3. Сайт по адресу `https://твой-ник.github.io/имя-репо`

## Логика игры

**Организатор до игры:**
- Добавляет каждое вино: название + правильные ответы по всем категориям
- Может редактировать категории (название, множитель, варианты, вкл/выкл)
- Запускает игру → получает 6-значный код

**Игрок:**
- Заходит по коду и имени
- На каждое вино: выбирает ответ и сумму ставки в любых категориях (можно пропускать)
- Жмёт «Поставить» → подтверждает на следующем экране → фишки списываются
- Видит ставки других игроков пока ждёт открытия ответов
- После открытия видит результаты с анимацией

**Организатор во время игры:**
- Кнопка «🔓 Открыть ответы» — все ответы по вину открываются сразу, фишки начисляются
- Кнопка «Дальше →» — переход к следующему вину
- Кнопка «⏹» — завершить игру с финальной таблицей

## Множители

- Угадал — возвращается `ставка × множитель`
- Не угадал — ставка теряется

Дефолтные множители: Часть света ×2, Страна ×3, Сорт ×2, Выдержка ×2, Год урожая ×3 (можно менять).

## Восстановление сессии

При перезагрузке страницы сессия восстанавливается автоматически (хранится в браузере). На главном экране показывается баннер «Продолжить» если есть активная сессия.

## Технологии

- **Хостинг**: GitHub Pages (статика)
- **БД**: Supabase (PostgreSQL + Realtime через WebSocket)
- **Фронтенд**: ванильный JS, никаких сборщиков
